# Mobile App Quality

## Goal

Keep mobile app changes safe, maintainable, and consistent across Expo, React Native bare, and Flutter projects.

## Quality Bar

- Reuse existing navigation, state, styling, component, and platform patterns.
- Preserve app architecture and public component/service APIs.
- Check iOS and Android differences when behavior may diverge.
- Handle loading, error, empty, offline, and permission states when relevant.
- Respect safe areas, keyboard behavior, scroll behavior, and small-screen layouts.
- Avoid dependency additions and native module changes unless justified.
- Avoid broad app config, signing, Gradle, Pod, or build changes without approval.
